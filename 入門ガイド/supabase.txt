最後にsupabaseに関して説明していきます。

無料版だと長い期間使用されていないと自動で閉じてしまうので定期的にアクションを与えるか直前に作成することをお勧めします。

filedetails.txtでfestival系はtitleを完全に一致させてくれと言われたと思いますがそれをsupabaseにもやっていきます。

1. supabaseに入り、table editorを選択してそこにあるstatusを選びます。
2. 上にあるInsertを選択してInsert rowを選びます。
3. idは空の状態、nameの部分にfestivalのtitleを入れます。それ以降は空でいいです。

例：
    { title: '1-1', attributes: [Sun], floor: 4, x: 1165, y: 500 },
    { title: '1-2', attributes: [Sun], floor: 4, x: 1085, y: 500 },
    { title: '1-3', attributes: [Sun], floor: 4, x: 935, y: 500 },

    などとあった場合

    それぞれ正確にinsertして

    id  name  status  ticket   numbered
    1   1-1   空      空        FALSE
    2   1-2   空      空        FALSE
    3   1-3   空      空        FALSE

    のようになっていればいいです。とにかくtitleとnameを一致させてください。
    ※idは1,2,3でなくてもいいです。（idは使わないので）例えば 1,4,12でもいいです。（自動生成されるので気にしなくていい）
    ※順番はどうでもいいです。(結局nameに一致するものを取得するようになっているので順番は関係ないです。) 例えば 1-1 と 1-2 がテーブル上で逆になっていても問題ない。

最後に自分のアカウントをsupabaseのauthにセットしていきます。
supabaseを開き、サイドバーからAuthenticationに飛びます。
Usersのページで一番右にadd userという部分があるのでクリックします。
そしてcreate new userを選択してそこに自分がサイトの管理画面で使用する予定のメールアドレスとパスワードを設定します。

これらのセットアップが完了したら晴れてすべての機能がフルに活用できるようになります。

ここまでのセットアップお疲れさまでした。特に問題なくここまで来れたでしょうか？比較的分かりやすいように細部まで説明したつもりでしたが分からないところや不明な点があったら自分で調べるなどして解決してください。

ではサイト作成がうまくいくことを願っています。

最後によく使うコマンドをおいておきます。

npm run dev
これはローカル環境を起動するためのコマンドです。
あとこれを打った後はbashターミナルは閉じないようにしてください。閉じちゃうと環境が落ちます。
